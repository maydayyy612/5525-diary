# 5525-DIARY

一个电影感黑金蓝红风格的私人演出记录档案馆，专为手机浏览设计。纯前端实现（HTML/CSS/JS + IndexedDB），无需后端服务器，可直接部署在 GitHub Pages 上。

## 功能

- **开屏页**：黑幕 + "5525 · 回到那一天"，轻触进入
- **首页时间轴**：所有演出记录按票根形式呈现，点击查看详情
- **记录场次**：演出名称 / 时间 / 票价 / 城市 / 场地 / 座位 / 评分 / 歌单（不限数量）/ 演出记忆点 / 现场照片（可从手机相册或相机上传）
- **编辑 / 删除**：详情页可编辑，编辑页内可删除
- **照片墙**：所有场次的照片汇总展示，点击可跳转回对应记录
- **人生地图**：中国地图轮廓（风格化呈现，非精确测绘），点亮去过的城市，可编辑交通方式与距离北京的公里数（默认按经纬度直线距离自动计算，可手动覆盖为实际交通距离）
- **搜索 / 筛选**：支持按歌曲名或城市名搜索

## 数据存储说明

所有数据（包括上传的照片）默认保存在**你当前手机 / 浏览器本地的 IndexedDB** 中，不会自动上传。现在已经加入了 **GitHub 云同步** 功能（见下），强烈建议开启，避免换手机、清缓存导致数据丢失。

## 云同步（把 GitHub 仓库当免费云端）

点击顶部栏右上角的 ☁ 图标，打开"云同步"面板。

### 第一步：生成一个 GitHub 令牌（Token）

1. 打开 [github.com/settings/personal-access-tokens/new](https://github.com/settings/personal-access-tokens/new)（Fine-grained personal access token）
2. Token name 随便填，比如 `5525-dairy-sync`
3. Expiration 建议设置一个具体期限（比如 90 天，到期后可以重新生成）
4. Repository access 选择 **Only select repositories**，只勾选 `5525-dairy` 这一个仓库
5. Permissions 里找到 **Contents**，设置为 **Read and write**
6. 生成后会显示一长串以 `github_pat_` 开头的字符串，**只显示一次**，复制保存好

### 第二步：在网站里填写设置

在"云同步"面板里填：
- GitHub 用户名
- 仓库名（`5525-dairy`）
- 分支（默认 `main`）
- 粘贴刚才生成的令牌
- 点击"保存设置"

### 第三步：推送 / 拉取

- **⬆ 推送到 GitHub**：把这台设备上的所有记录和照片上传到仓库的 `data/shows.json`、`data/cities.json` 和 `photos/` 目录
- **⬇ 从 GitHub 拉取**：把云端最新的数据拉回来，覆盖这台设备本地的数据

推荐用法：每次在某台设备上新增/修改完记录后，点一次"推送"；换到另一台设备时，先点一次"拉取"再开始用，避免互相覆盖。

### 关于安全

- 令牌只保存在**你这台设备当前浏览器**的本地存储里，不会上传到除 GitHub 之外的任何地方，也不会被写进网站代码或仓库文件里
- 一定要用上面说的 **Fine-grained token**，只授权这一个仓库、只给 Contents 读写权限 —— 这样即使令牌泄露，影响范围也只限于这一个仓库
- 如果是在公共电脑或者别人的手机上用过，用完记得点"清除本机令牌"
- 令牌到期或者怀疑泄露了，随时可以去 GitHub 设置页里删除旧令牌、生成新的

### 关于照片

照片上传后会变成仓库 `photos/` 目录下的图片文件（不再是又大又占地方的 base64 文本），因为这些图片文件也会被 GitHub Pages 一起部署，所以在**部署好的网站地址**下能直接正常显示；如果你在本地直接打开 `index.html` 预览（还没部署过的文件），拉取下来的记录里图片可能显示不出来，这是正常的，等推送/部署完成后用正式网址访问即可。

## 文件结构

```
5525-dairy/
├── index.html      页面结构与所有弹层
├── style.css        电影黑金蓝红主题样式
├── map-data.js       中国地图轮廓 + 城市经纬度 + 距离计算
├── db.js             IndexedDB 数据封装 + 图片压缩
├── sync.js           GitHub 云同步逻辑
├── app.js            页面渲染与交互逻辑
└── README.md
```

## 部署到 GitHub Pages

1. 打开你的仓库 `5525-dairy`
2. 删除仓库里旧的文件，把这个文件夹里的所有文件（index.html、style.css、map-data.js、db.js、sync.js、app.js、README.md）上传 / push 到仓库根目录
3. 进入仓库 Settings → Pages，Source 选择 `main` 分支、根目录 `/ (root)`，保存
4. 等待一两分钟，用手机浏览器打开 `https://你的用户名.github.io/5525-dairy/`，建议"添加到主屏幕"当作 App 使用

## 关于地图

地图轮廓是风格化的简化图形，用于氛围呈现，不是精确测绘边界；城市点位置基于经纬度换算，距离北京的公里数则是通过经纬度实时计算的真实直线距离，数值是准确的。

如果你去过的城市不在内置的城市库里（`map-data.js` 里的 `CITY_COORDS`），地图上暂时不会显示光点，但记录本身完全不受影响，城市列表里依然会正常显示，你也可以告诉我城市名，我帮你把坐标加进去。
