# 5525-DIARY

一个电影感黑金蓝红风格的私人演出记录档案馆，专为手机浏览设计。纯前端实现（HTML/CSS/JS + IndexedDB），无需后端服务器，可直接部署在 GitHub Pages 上。

## 功能

- **开屏页**：黑幕 + "5525 · 回到那一天"，轻触进入
- **首页时间轴**：所有演出记录按票根形式呈现，点击查看详情
- **记录场次**：演出名称 / 时间 / 票价 / 城市 / 场地 / 座位 / 评分 / 歌单（不限数量）/ 演出记忆点 / 现场照片（可从手机相册或相机上传）
- **编辑 / 删除**：详情页可编辑，编辑页内可删除
- **照片墙**：所有场次的照片汇总展示，点击可跳转回对应记录
- **人生地图**：中国地图轮廓（风格化呈现，非精确测绘），点亮去过的城市，可编辑交通方式与距离北京的公里数（默认按经纬度直线距离自动计算，可手动覆盖为实际交通距离）
- **搜索 / 筛选**：支持按歌曲名或城市名搜索

## 数据存储说明 ⚠️

所有数据（包括上传的照片）保存在**你当前手机 / 浏览器本地的 IndexedDB** 中，不会上传到任何服务器，也不会在不同设备间自动同步。也就是说：

- 如果你换手机、换浏览器，或清除浏览器数据，之前记录的内容会丢失，请注意不要清理该网站的浏览器数据
- 目前没有跨设备同步功能；如果你需要经常换设备使用，可以告诉我，我可以帮你加一个"导出 / 导入 JSON 备份"的功能，或接入一个免费的云同步方案
- 建议偶尔手动截图或备份重要记录，作为保险

## 部署到 GitHub Pages

1. 打开你的仓库 `5525-dairy`
2. 删除仓库里旧的文件，把这个文件夹里的所有文件（index.html、style.css、map-data.js、db.js、app.js、README.md）上传 / push 到仓库根目录
3. 进入仓库 Settings → Pages，Source 选择 `main` 分支、根目录 `/ (root)`，保存
4. 等待一两分钟，用手机浏览器打开 `https://你的用户名.github.io/5525-dairy/`，建议"添加到主屏幕"当作 App 使用

## 关于地图

地图轮廓是风格化的简化图形，用于氛围呈现，不是精确测绘边界；城市点位置基于经纬度换算，距离北京的公里数则是通过经纬度实时计算的真实直线距离，数值是准确的。

如果你去过的城市不在内置的城市库里（`map-data.js` 里的 `CITY_COORDS`），地图上暂时不会显示光点，但记录本身完全不受影响，城市列表里依然会正常显示，你也可以告诉我城市名，我帮你把坐标加进去。
